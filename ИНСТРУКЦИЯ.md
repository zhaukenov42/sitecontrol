# SiteControl — Инструкция по доработке и публикации

**Проект:** Construction Monitoring & Recovery Platform  
**Стек:** React 18 + TypeScript + Vite + Tailwind CSS (фронтенд) · Node.js + Express + sql.js (бэкенд)  
**Дата сборки:** 16 июня 2026

---

## 1. Установка и запуск на новом компьютере

### Требования
- [Node.js](https://nodejs.org/) версии **18 или новее** (рекомендуется LTS)
- npm (идёт вместе с Node.js)

### Шаги

```bash
# 1. Распакуй архив и перейди в папку проекта
cd sitecontrol

# 2. Установи зависимости (один раз)
npm install

# 3. Запусти проект в режиме разработки
npm run dev
```

После запуска открой в браузере:
- **Приложение:** http://localhost:5173
- **API:** http://localhost:3001

> База данных создаётся автоматически при первом запуске (`server/sitecontrol.db.bin`).  
> Если нужно сбросить данные к демо-состоянию — удали этот файл и перезапусти сервер.

## 3. Ключевые точки доработки

### 3.1 Добавить новую WBS-зону или риск

**Через UI:** страницы уже поддерживают добавление.  
**Через код (сидовые данные):** `server/src/db.ts` — найди массивы `wbs_zones` и `risks`, добавь строки.

---

### 3.2 Изменить пороговое значение SPI для Recovery Mode

Файл: `server/src/routes/project.ts`  
Найди строку с `spi < 0.85` и измени порог.

Также обнови на фронтенде: `client/src/components/RecoveryBanner.tsx`

---

### 3.3 Добавить новый раздел в план восстановления

Файл: `client/src/pages/RecoveryPlanPage.tsx`  
В массиве `sections` добавь новый объект:
```ts
{ key: 'new_section', label: 'Название раздела', content: '' }
```

---

### 3.4 Подключить реальный импорт P6 (XER/XLSX)

Сейчас импорт генерирует тестовые данные. Для реального парсинга:

1. Установи библиотеку: `npm install xlsx --workspace=server`
2. Файл: `server/src/routes/schedule.ts` — замени mock-логику на парсинг через `xlsx.readFile()`
3. XER-формат (нативный Primavera) потребует отдельного парсера —  
   рекомендуется библиотека `p6-xer-parser` (npm) или ручной парсинг TSV-структуры XER.

---

### 3.5 Генерация DOCX/PDF для плана восстановления

Сейчас кнопки "Экспорт" — заглушки. Для реализации:

```bash
npm install docx pdfkit --workspace=server
```

Файл: `server/src/routes/recoveryPlans.ts`  
Добавь маршрут `GET /api/recovery-plans/:id/export/docx` используя библиотеку `docx`,  
и `GET /api/recovery-plans/:id/export/pdf` используя `pdfkit`.

---

### 3.6 Добавить аутентификацию

Сейчас приложение без авторизации. Для добавления:

1. Установи: `npm install jsonwebtoken bcrypt --workspace=server`
2. Добавь таблицу `users` в `db.ts`
3. Создай маршруты `POST /api/auth/login` и `POST /api/auth/register`
4. Защити все маршруты middleware-ом `verifyToken`
5. На фронтенде — форма логина + хранение токена в `localStorage`

---

### 3.7 Изменить язык интерфейса

Все строки интерфейса находятся напрямую в компонентах.  
Для билингвальности (RU/EN) рекомендуется подключить `i18next`:

```bash
npm install i18next react-i18next --workspace=client
```

## 4. Переход на продакшн-базу данных (PostgreSQL)

Текущая БД — `sql.js` (SQLite в памяти). Для продакшна:

1. Установи: `npm install pg --workspace=server`
2. В `server/src/db.ts` замени sql.js на `pg.Pool`
3. Создай базу данных PostgreSQL и перенеси схему из `db.ts`
4. Добавь переменную окружения `DATABASE_URL`

## 5. Публикация (Деплой)

### Вариант A — Быстрый старт: Railway.app (рекомендуется)

1. Зарегистрируйся на [railway.app](https://railway.app)
2. Подключи GitHub-репозиторий
3. Создай два сервиса: **server** и **client**
4. Укажи переменные окружения:
   - `NODE_ENV=production`
   - `PORT=3001` (для сервера)
   - `VITE_API_URL=https://твой-сервер.railway.app` (для клиента)
5. Railway автоматически задеплоит при каждом `git push`

**Стоимость:** ~$5/мес (Hobby план)

---

### Вариант B — VPS (полный контроль)

#### Требования к серверу
- Ubuntu 22.04, минимум 1 CPU / 1 GB RAM
- Порты 80 и 443 открыты

#### Установка

```bash
# На сервере
sudo apt update && sudo apt install -y nodejs npm nginx certbot

# Клонируй репо и установи зависимости
git clone <repo-url> /var/www/sitecontrol
cd /var/www/sitecontrol && npm install

# Собери фронтенд
npm run build --workspace=client

# Запусти сервер через PM2
npm install -g pm2
pm2 start server/src/index.ts --name sitecontrol-api --interpreter ts-node
pm2 save && pm2 startup
```

#### Nginx конфиг (`/etc/nginx/sites-available/sitecontrol`)

```nginx
server {
    listen 80;
    server_name твой-домен.kz;

    # Фронтенд (статика)
    location / {
        root /var/www/sitecontrol/client/dist;
        try_files $uri $uri/ /index.html;
    }

    # API
    location /api/ {
        proxy_pass http://localhost:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

```bash
# Включи сайт и получи SSL
sudo ln -s /etc/nginx/sites-available/sitecontrol /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
sudo certbot --nginx -d твой-домен.kz
```

### Вариант C — Docker (для команды)

Создай `docker-compose.yml` в корне проекта:

```yaml
version: '3.9'
services:
  server:
    build: ./server
    ports:
      - "3001:3001"
    environment:
      - NODE_ENV=production
    volumes:
      - db-data:/app/data

  client:
    build: ./client
    ports:
      - "80:80"
    depends_on:
      - server

volumes:
  db-data:
```

```bash
docker-compose up -d
```

## 6. Переменные окружения

Создай файл `server/.env` для настройки:

```env

NODE_ENV=development
DB_PATH=./sitecontrol.db.bin
JWT_SECRET=замени-на-случайную-строку
SMTP_HOST=smtp.gmail.com
SMTP_USER=твой@email.com
SMTP_PASS=пароль-приложения
```

Создай `client/.env`:

```env
VITE_API_URL=http://localhost:3001
```
